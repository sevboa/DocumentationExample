Описание работы микросервиса

Первым потоком Микросервис забирает сообщения из топика Kafka в рамках метода получения сообщений и сохраняет курсы из сообщения в свою базу.

Вторым потоком микросервис ожидает входящего запроса от потребителя и предоставляет данные по курсам из своей базы.

Метод получения сообщений из топика Kafka currency-rate-from-ZAGAR

  1. Получает сообщение из топика Kafka с именем согласно конфигурационному параметру kafka_topic и форматом согласно формату сообщения очереди

  2. Сообщение от ZAGAR делятся на отдельные курсы и сортируются по limit от меньшего к большему

  3. Происходит обработка каждого курса в отдельности

    1. Проводится поиск совпадающего по дате от, валюте, группе продукта, лимиту и офисам курса Запрос поиска совпадающего курса

    2. Если такой курс ненайден, то:

      1. Проводится поиск предыдущего курса и замена даты до на дату от согласно Запрос поиска старого курса и заполнение даты до датой от нового курса

      2. Если limit > 0, то выполняется Запрос поиска лимита от и добавление лимита до

      3. Выполняется Запрос добавления нового курса

  4. В топик передается подтверждение о том что полученное сообщение было обработано

  5. Конец работы метода

Метод поиска актуальных курсов по параметрам

  1. Выполняется проверка параметров запроса на соответствие протоколу согласно контракту микросервиса

    1. Если проверка не успешна

      1. Выполняется альтернативный сценарий EC-Validation

  2. На основе входящих параметров формируется SQL запрос поиска курсов

  3. SQL запрос поиска курсов выполняется

  4. Микросервис преобразовывает результат согласно преобразованию результата SQL запроса поиска курсов T_01_01

  5. Микросервис передает исходящий JSON объект REST сервиса потребителю

  6. Конец работы метода

Запрос поиска совпадающего курса

SELECT EGAR_CURRENCY_RATE.ID IS NOT NULL
FROM EGAR_CURRENCY_RATE
INNER JOIN OFFICE_CODE_EGAR_CURRENCY_RATE ON EGAR_CURRENCY_RATE.ID = OFFICE_CODE_EGAR_CURRENCY_RATE.EGARCURRENCYRATE_ID
WHERE 
    EGAR_CURRENCY_RATE.EGARPRODUCTGROUP_CODE = $productGroupCode
    AND EGAR_CURRENCY_RATE.EXCHANGERATELIMITFROM = $limit
    AND OFFICE_CODE_EGAR_CURRENCY_RATE.OFFICECODE IN $officeCode
    AND EGAR_CURRENCY_RATE.BASECURRENCYSIGNCODE = $baseCurrencySignCode
    AND EGAR_CURRENCY_RATE.QUOTEDCURRENCYSIGNCODE = $quotedCurrencySignCode
    AND EGAR_CURRENCY_RATE.VALIDFROM = $validFrom

Запрос поиска старого курса и заполнение даты до

UPDATE EGAR_CURRENCY_RATE
SET EGAR_CURRENCY_RATE.VALIDTO = $validFrom
WHERE
    EGAR_CURRENCY_RATE.ID IN (
        SELECT EGAR_CURRENCY_RATE.ID
        FROM EGAR_CURRENCY_RATE
        INNER JOIN OFFICE_CODE_EGAR_CURRENCY_RATE ON EGAR_CURRENCY_RATE.ID = OFFICE_CODE_EGAR_CURRENCY_RATE.EGARCURRENCYRATE_ID
        WHERE 
            EGAR_CURRENCY_RATE.EGARPRODUCTGROUP_CODE = $productGroupCode
            AND OFFICE_CODE_EGAR_CURRENCY_RATE.OFFICECODE IN $officeCode
            AND EGAR_CURRENCY_RATE.BASESIGNCURRENCYCODE = $baseCurrencySignCode
            AND EGAR_CURRENCY_RATE.QUOTEDSIGNCURRENCYCODE = $quotedCurrencySignCode
            AND EGAR_CURRENCY_RATE.VALIDTO IS NULL
        ORDER BY VALIDFROM DESC
    )
;

Запрос поиска лимита от и добавление лимита до

UPDATE EGAR_CURRENCY_RATE
SET EGAR_CURRENCY_RATE.EXCHANGERATELIMITTO = $limit
WHERE
    EGAR_CURRENCY_RATE.ID IN (SELECT EGAR_CURRENCY_RATE.ID
    FROM EGAR_CURRENCY_RATE
    INNER JOIN OFFICE_CODE_EGAR_CURRENCY_RATE ON OFFICE_CODE_EGAR_CURRENCY_RATE.EGARPRODUCTGROUP_ID = EGAR_CURRENCY_RATE.ID
    WHERE 
        EGAR_CURRENCY_RATE.EGARPRODUCTGROUP_CODE = $productGroupCode
        AND EGAR_CURRENCY_RATE.EXCHANGERATELIMITFROM < $limit
        AND EGAR_CURRENCY_RATE.EXCHANGERATELIMITTO IS NULL
        AND OFFICE_CODE_EGAR_CURRENCY_RATE.OFFICECODE IN $officeCode
        AND EGAR_CURRENCY_RATE.BASESIGNCURRENCYCODE = $baseCurrencySignCode
        AND EGAR_CURRENCY_RATE.QUOTEDSIGNCURRENCYCODE = $quotedCurrencySignCode
        AND EGAR_CURRENCY_RATE.VALIDFROM = $validFrom
        )
    ORDER BY EGAR_CURRENCY_RATE.EXCHANGERATELIMITFROM ASC)
;

Запрос поиска курса

SELECT
    CURRENCY_RATE_TYPE.CODE AS 'currencyRateTypeCode',
    ARRAY(
        SELECT OFFICE_CODE_EGAR_CURRENCY_RATE.OFFICECODE
        FROM OFFICE_CODE_EGAR_CURRENCY_RATE
        WHERE OFFICE_CODE_EGAR_CURRENCY_RATE.EGARCURRENCYRATE_ID = EGAR_CURRENCY_RATE.ID
        ORDER BY OFFICE_CODE_EGAR_CURRENCY_RATE.OFFICECODE ASC
        ) AS 'bankOfficeCodes',
    EGAR_CURRENCY_RATE.EXCHANGERATELIMITFROM AS 'exchangeRateLimitFrom',
    EGAR_CURRENCY_RATE.EXCHANGERATELIMITTO AS 'exchangeRateLimitTo',
    EGAR_CURRENCY_RATE.BASECURRENCYSIGNCODE AS 'baseCurrencySignCode',
    EGAR_CURRENCY_RATE.BASECURRENCYDIGITCODE AS 'baseCurrencyDigitCode',
    EGAR_CURRENCY_RATE.QUOTEDCURRENCYSIGNCODE AS 'quotedCurrencySignCode',
    EGAR_CURRENCY_RATE.QUOTEDCURRENCYDIGITCODE AS 'quotedCurrencyDigitCode',
    EGAR_CURRENCY_RATE.BASECURRENCYSCALE AS 'baseCurrencyScale',
    EGAR_CURRENCY_RATE.QUOTEDCURRENCYSCALE AS 'quotedCurrencyScale',
    EGAR_CURRENCY_RATE.BUYRATE AS 'buyRate',
    EGAR_CURRENCY_RATE.SELLRATE AS 'sellRate',
    EGAR_CURRENCY_RATE.VALIDFROM AS 'validFrom'
FROM EGAR_CURRENCY_RATE
INNER JOIN EGAR_PRODUCT_GROUP ON EGAR_PRODUCT_GROUP.CODE = EGAR_CURRENCY_RATE.EGARPRODUCTGROUP_CODE
INNER JOIN CURRENCY_RATE_TYPE ON CURRENCY_RATE_TYPE.ID = EGAR_PRODUCT_GROUP.CURRENCYRATETYPE_ID
-- если в запросе заполнено поле bankOfficeCodes
    INNER JOIN OFFICE_CODE_EGAR_CURRENCY_RATE ON OFFICE_CODE_EGAR_CURRENCY_RATE.EGARCURRENCYRATE_ID = EGAR_CURRENCY_RATE.ID
WHERE
    CURRENCY_RATE_TYPE.CODE = $currencyRateTypeCode
    AND EGAR_CURRENCY_RATE.VALIDFROM <= NOW()
    AND EGAR_CURRENCY_RATE.VALIDTO > NOW()
    -- если в запросе заполнено поле bankOfficeCodes
        AND (
            OFFICE_CODE_EGAR_CURRENCY_RATE.CODE IN $bankOfficeCodes
            OR OFFICE_CODE_EGAR_CURRENCY_RATE.CODE = '0'
        )
    -- если в запросе заполнено поле currencyExchangeAmount
        AND EGAR_CURRENCY_RATE.EXCHANGERATELIMITFROM <= $currencyExchangeAmount
        AND (
            EGAR_CURRENCY_RATE.EXCHANGERATELIMITTO > $currencyExchangeAmount
            OR EGAR_CURRENCY_RATE.EXCHANGERATELIMITTO IS NULL
        )
    -- если в запросе заполнено поле baseCurrencySignCode
        AND EGAR_CURRENCY_RATE.BASECURRENCYSIGNCODE = $baseCurrencySignCode
    -- если в запросе заполнено поле baseCurrencyDigitCode
        AND EGAR_CURRENCY_RATE.BASECURRENCYDIGITCODE = $baseCurrencyDigitCode
    -- если в запросе заполнено поле quotedCurrencySignCode
        AND EGAR_CURRENCY_RATE.QUOTEDCURRENCYSIGNCODE = $quotedCurrencySignCode
    -- если в запросе заполнено поле quotedCurrencyDigitCode
        AND EGAR_CURRENCY_RATE.QUOTEDCURRENCYDIGITCODE = $quotedCurrencyDigitCode
GROUP BY EGAR_CURRENCY_RATE.ID

    1. Способ логирования: стандартный механизм

Метрики

  1. В процессе работы с момента запуска сервис должен собирать RED метрики:

    • Rate, Error - count (кол-во запросов)

    • Duration - max (максимальное время), totalTime (общее время всех запросов) и count (кол-во запросов). Среднее можно вычислить на стороне системы мониторинга, как totalTime/count

    • Rate, Error, Duration собираем по каждой операции (методу) микросервиса

    • Duration собираем для вызовов внешних бэковых интерфейсов (JDBC, WS, MQ). Max, totalTime и count
